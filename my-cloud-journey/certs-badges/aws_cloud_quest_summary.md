I completed the **AWS Cloud Quest: Cloud Practitioner** role on AWS Skill Builder, earning the official digital badge that certifies hands-on foundational cloud skills. Cloud Quest is Amazon Web Services' role-playing game (RPG)-style learning experience where learners solve real-world cloud challenges by building practical solutions on AWS.

## What I Learned

### ☁️ Cloud Fundamentals
- What cloud computing is and the key benefits: scalability, elasticity, cost savings, and global reach
- The difference between **on-premises**, **cloud**, and **hybrid** infrastructure
- Core AWS global infrastructure: **Regions**, **Availability Zones**, and **Edge Locations**

### 🖥️ Compute
- Launching and configuring **Amazon EC2** (Elastic Compute Cloud) virtual machines
- Understanding instance types, AMIs (Amazon Machine Images), and pricing models (On-Demand, Reserved, Spot)
- Introduction to **AWS Lambda** for serverless, event-driven computing

### 🗄️ Storage
- Working with **Amazon S3** (Simple Storage Service) for object storage
- Understanding S3 storage classes (Standard, Infrequent Access, Glacier) and lifecycle policies
- Introduction to **Amazon EBS** (Elastic Block Store) for persistent block-level storage

### 🌐 Networking
- Understanding **Amazon VPC** (Virtual Private Cloud): subnets, route tables, and internet gateways
- How **Security Groups** and **Network ACLs** control inbound/outbound traffic
- Basics of **Amazon Route 53** for DNS and traffic routing

### 🗃️ Databases
- Overview of **Amazon RDS** (Relational Database Service) and managed relational databases
- Introduction to **Amazon DynamoDB** for NoSQL, key-value data storage
- When to choose relational vs. NoSQL databases

### 🔐 Security & Identity
- Core security principles on AWS: **shared responsibility model**
- Managing access with **AWS IAM** (Identity and Access Management): users, groups, roles, and policies
- Principle of **least privilege** and how to apply it

### 💰 Billing & Pricing
- Understanding the **AWS Free Tier** and how to avoid unexpected charges
- Using the **AWS Pricing Calculator** and **Cost Explorer**
- Overview of **AWS Support Plans**

### 🛠️ Core Services Summary

| Service | Category | Purpose |
|---|---|---|
| Amazon EC2 | Compute | Virtual servers in the cloud |
| AWS Lambda | Compute | Serverless function execution |
| Amazon S3 | Storage | Scalable object storage |
| Amazon EBS | Storage | Block storage for EC2 |
| Amazon RDS | Database | Managed relational databases |
| Amazon DynamoDB | Database | NoSQL key-value store |
| Amazon VPC | Networking | Isolated cloud network |
| AWS IAM | Security | Access and identity management |
| Amazon Route 53 | Networking | DNS and traffic routing |
| AWS CloudWatch | Monitoring | Metrics, logs, and alarms |

---

## Key Takeaways

- Cloud computing shifts infrastructure from a **capital expense (CapEx)** to an **operational expense (OpEx)** model
- AWS operates on a **pay-as-you-go** pricing model — you only pay for what you use
- The **shared responsibility model** means AWS secures the cloud infrastructure, while customers are responsible for securing their data and configurations within it
- Real-world cloud problems are best solved by combining multiple AWS services together
